# ChestX-ray14 Diagnosis Assistant

AI-powered decision-support tool that analyzes chest X-rays to detect 14 disease
findings, built on the NIH ChestX-ray14 dataset. Uses a DenseNet121 backbone with
transfer learning for multi-label classification, and Grad-CAM for explainability.

**This is a research/decision-support prototype — not a certified diagnostic
device.** See [Limitations](#limitations) below.

## Results

Trained on the full ~112k-image NIH ChestX-ray14 dataset with a patient-disjoint
train/val/test split.

| Metric | Value |
|---|---|
| Validation macro-AUC-ROC | 0.77 |
| Test macro-AUC-ROC | 0.778 |
| Test macro F1 (default threshold) | 0.263 |
| Test macro F1 (optimal threshold) | 0.234 |

**Per-class test results:**

| Class | AUC-ROC | F1-Score |
|---|---|---|
| Cardiomegaly | 0.845 | 0.361 |
| Emphysema | 0.863 | 0.356 |
| Hernia | 0.882 | 0.022 |
| Pneumothorax | 0.841 | 0.443 |
| Edema | 0.832 | 0.226 |
| Effusion | 0.796 | 0.447 |
| Fibrosis | 0.797 | 0.127 |
| Mass | 0.767 | 0.256 |
| Atelectasis | 0.745 | 0.336 |
| Nodule | 0.717 | 0.234 |
| Pleural_Thickening | 0.736 | 0.182 |
| Consolidation | 0.705 | 0.209 |
| Infiltration | 0.684 | 0.435 |
| Pneumonia | 0.685 | 0.049 |

AUC-ROC is consistently strong across classes (0.68–0.88). F1-scores are lower
and more variable — expected for a severely imbalanced multi-label task at a
uniform decision threshold; classes with very few positive examples (e.g.
Hernia) score low on F1 despite reasonable AUC, since F1 is threshold-sensitive
in a way AUC is not. Note the optimal (Youden's J) threshold gives a *lower*
macro F1 than the default 0.5 threshold here — Youden's J balances sensitivity
and specificity, not F1 directly, so it isn't guaranteed to maximize F1.

Full per-class breakdown (Accuracy, Precision, Recall, Specificity, at both
thresholds) is in `outputs/metrics/metrics.csv`, generated by the notebook.

Training curves, confusion matrices, ROC curves, and Grad-CAM visualizations are
also produced by the notebook and saved under `outputs/`.

## Approach

- **Data**: NIH ChestX-ray14 (224×224 resized), 14 disease labels, multi-label
  (an image can have multiple findings). Split via the official
  `train_val_list_NIH.txt` / `test_list_NIH.txt`, with `train_val` further split
  into train/val **by Patient ID** (80/20) to prevent the same patient's scans
  appearing in both sets.
- **Preprocessing**: MONAI transform pipeline — grayscale to 3-channel, resize,
  ImageNet normalization. Training augmentation (horizontal flip, small rotation,
  contrast jitter, mild noise) chosen specifically for chest X-rays — no vertical
  flip, since that would invert real anatomy.
- **Model**: DenseNet121, ImageNet-pretrained, classifier head replaced for
  14-way multi-label output. Two-phase training — phase 1 trains only the new
  classifier head (backbone frozen), phase 2 fine-tunes the full network at a
  lower learning rate.
- **Class imbalance**: `BCEWithLogitsLoss` with per-class `pos_weight` computed
  from train-split frequencies, capped to prevent the rarest classes from
  destabilizing training.
- **Training**: mixed precision (`torch.amp`), `AdamW`, `ReduceLROnPlateau` on
  validation macro-AUC, early stopping, checkpointing, TensorBoard logging.
- **Evaluation**: per-class + macro Accuracy / Precision / Recall / Specificity /
  F1 / AUC-ROC, at both a default 0.5 threshold and a per-class optimal
  (Youden's J) threshold, confusion matrices, ROC curves.
- **Explainability**: Grad-CAM (target layer `features.norm5`), overlaid on
  original X-rays, cross-checked against NIH's ground-truth bounding boxes
  where available.

## Setup & Usage

1. **Open in Google Colab** with a GPU runtime (Runtime → Change runtime type →
   GPU).
2. **Kaggle credentials** (one-time): get an API token from
   [kaggle.com](https://www.kaggle.com) → Account → Create New API Token. In
   Colab, add `KAGGLE_USERNAME` and `KAGGLE_KEY` as Secrets (key icon, left
   sidebar), enable notebook access.
3. **Google Drive**: the notebook mounts your Drive and stores the dataset and
   all outputs there so they persist across sessions.
4. **Run the cells in order**, top to bottom. Cell 1 installs dependencies and
   checks the environment; Cell 2 downloads the dataset; Cells 3–7 handle
   exploration, splitting, and model setup; Cell 8 trains (the longest step);
   Cells 9–11 evaluate, generate explainability visuals, and demo inference.

## Limitations

- **Not a diagnostic device** — a research/decision-support prototype only.
  Never use for actual patient diagnosis or treatment without independent
  clinical validation and regulatory clearance.
- **Label noise**: ChestX-ray14's labels were extracted via NLP from radiology
  reports, not verified per-image by radiologists.
- **Single-institution data**: all data comes from one hospital system;
  generalization to other populations/scanners is unverified (no external
  validation was performed).
- **Grad-CAM caveats**: heatmaps show what influenced a prediction, not proof
  the model reasoned correctly — a debugging/transparency aid, not a
  correctness guarantee.

Full discussion of design decisions, limitations, and ethical considerations is
in the notebook's final markdown cell.
